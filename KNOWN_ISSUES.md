# Known Issues

这个文件只写当前仓库已经确认的边界和限制。

## 当前定位

这个仓库当前适合：

- 粗剪
- 初版装配
- 节奏规划
- 素材筛选
- 可检查的第一版时间线生成

这个仓库当前不适合：

- 无人值守最终成片
- 稳定精剪
- 复杂关键帧动画自动化
- 稳定批量特效处理

## 已确认的限制

### 1. 关键帧动画还不稳定

当前关键帧相关能力还不能被理解成“可靠完成精细动画”。

更具体地说，目前还不能把它当成：

- AI 自动生成完整关键帧动画的工具
- 自动拉贝塞尔曲线、缓入缓出或速度曲线的工具
- 可以直接替代人工做精细动画调整的工具

实际影响：

- 某些关键帧写入可以成功
- 但复杂动画、缓动、图形曲线细节仍不稳定
- AI 现在更适合生成动作思路、镜头节奏和手工调整清单
- 最终仍需要在 Premiere 里人工复核

### 2. 转场整体已经可用，但仍建议显式指定

转场能力现在整体成功率已经比较高，尤其在明确指定转场类型时更稳。

实际影响：

- 常见转场在自动装配里通常可以正常写入
- 不同工程状态下，同样的转场调用仍可能出现个别差异
- 自动装配时仍然更适合显式指定转场，而不是完全依赖默认推断

### 3. 特效参数写入仍可能漂移或表现异常

实际影响：

- 特效本身多数情况下可以正常挂载
- 真正不稳的主要是特效参数，尤其是复杂参数、不同素材类型和不同工程状态下
- 参数写入可能出现漂移、数值不自然，或者和预期视觉结果不一致
- 这类能力更适合做辅助，而不是最终交付依赖

### 4. CEP 面板必须处于启动状态

如果 Premiere 里的 `PR MCP` 面板没有启动，MCP 客户端虽然能看到工具，但调用会失败。

常见症状：

- 工具列表能看到
- 但实际执行超时
- 或者没有拿到宿主返回结果

排查顺序：

1. 打开 `Window > Extensions > PR MCP`
2. 确认桥接目录是 `C:/pr-mcp-cmd`
3. 点击 `启动桥接`
4. 点击 `测试连接`

### 5. Node 侧和 CEP 侧桥接目录必须一致

当前默认桥接目录是：

```text
C:/pr-mcp-cmd
```

如果 `PREMIERE_TEMP_DIR` 和面板中的桥接目录不一致，就会出现“客户端能连上，但请求和响应文件不在一个目录”的问题。

### 6. 素材目录不是可选项

这个仓库的粗剪工作流依赖本地素材目录。

如果不给素材目录：

- AI 不知道有哪些素材
- 无法安全扫描素材
- 无法可靠规划镜头和时间线

### 7. 最终结果仍需人工复核

当前仓库更像“AI 辅助粗剪系统”，不是“全自动交付系统”。

你仍然需要人工检查：

- 镜头顺序
- 节奏
- 字幕
- 转场
- 关键帧
- 特效参数

## 常见误解

### “能控制 Premiere” 不等于 “能替代剪辑师”

这个仓库已经能通过 MCP 工具驱动 Premiere 做不少自动化动作，但这不等于它已经可以稳定完成最终交付。

### “能做粗剪” 不等于 “能稳定做精剪”

当前最稳的价值是：

- 把第一版时间线搭出来
- 帮你减少重复性装配工作
- 帮你把需求、素材和计划串起来

## 建议用法

最稳的工作方式是：

1. 给 AI 素材目录
2. 给 AI Word 文档 / 参考视频 / 提示词
3. 让 AI 先生成计划
4. 你确认计划
5. 再执行 Premiere 粗剪
6. 最后人工精修
